# Homelab / Self-Hosted Infrastructure

**Timeframe:** 2024 â€“ Present  
**Environment:** Ubuntu Server, Docker, Docker Compose  

## Objective
Build and maintain a secure, reliable self-hosted environment to practice system administration, networking, and service management.

## Architecture
- Linux-based host running Docker
- Services orchestrated with Docker Compose
- Private internal networking
- Persistent volumes for data durability

## Services Deployed
- Nextcloud (file storage)
- Uptime Kuma (service monitoring)
- Heimdall (service dashboard)
- Supporting databases and utilities

## Security Considerations
- Least-privilege container permissions
- Regular system and container updates
- No unnecessary public port exposure
- Firewall and access control enforcement

## Outcomes
- Improved understanding of Linux services and networking
- Real-world troubleshooting of availability and configuration issues
- Experience documenting infrastructure for reproducibility
